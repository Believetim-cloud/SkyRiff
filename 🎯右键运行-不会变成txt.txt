❌ 问题：双击 start.bat 会变成编辑文本
✅ 解决：用右键运行！


═══════════════════════════════════════════════════════════════


方法1：右键运行（最简单！）⭐⭐⭐⭐⭐
─────────────────────────────────────────────────────────────────

步骤：
1. 找到 server/start.bat 文件
2. 右键点击它
3. 选择 "运行" 或 "Run" 或 "打开"
4. 完成！


如果没有"运行"选项，选择：
  → "打开方式"
  → "命令提示符"
  → 或 "Windows 命令处理程序"


═══════════════════════════════════════════════════════════════


方法2：拖到 cmd 窗口（也很简单！）⭐⭐⭐⭐
─────────────────────────────────────────────────────────────────

步骤：
1. 按 Win+R，输入 cmd，回车
2. 把 start.bat 文件拖到 cmd 窗口
3. 按回车
4. 完成！


═══════════════════════════════════════════════════════════════


方法3：在文件夹地址栏输入 cmd（超快！）⭐⭐⭐⭐⭐
─────────────────────────────────────────────────────────────────

步骤：
1. 打开 server 文件夹
2. 点击顶部地址栏（显示路径的地方）
3. 输入：cmd
4. 按回车
5. 在弹出的命令行中输入：start.bat
6. 按回车
7. 完成！


这个方法最快最准确！


═══════════════════════════════════════════════════════════════


🔧 修复文件关联（一劳永逸）
─────────────────────────────────────────────────────────────────

如果您想让双击直接运行，修复一下文件关联：

步骤：
1. 右键 start.bat → 属性
2. 点击 "更改" 按钮（在"打开方式"旁边）
3. 选择 "Windows 命令处理程序" 或 "cmd.exe"
4. 勾选 "始终使用此应用打开 .bat 文件"
5. 确定
6. 以后双击就能直接运行了！


═══════════════════════════════════════════════════════════════


💡 为什么会变成编辑？
─────────────────────────────────────────────────────────────────

可能原因：
• .bat 文件被关联到了文本编辑器
• 某些安全软件修改了默认行为
• 系统设置问题


═══════════════════════════════════════════════════════════════

  2. 方法1（右键运行）
  3. 方法2（拖到cmd）
  4. 修复文件关联（一劳永逸）
